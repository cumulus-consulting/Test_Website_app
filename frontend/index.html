Architecture Overview

The solution is built using a multi-layered architecture to ensure scalability, security, and maintainability. Key components include:

  • Web Application Tier: Containerized frontend (HTML, CSS, Bootstrap, and JavaScript) and backend (Node.js) deployed on a Kubernetes cluster for a responsive and interactive user interface.
  • Database Layer: A MongoDB server hosted on a dedicated EC2 instance with authentication, configured for secure access by the Kubernetes cluster.
  • MongoDB Backups: Periodic MongoDB backups taken daily at 9am UTC using a bash script on the VM, then pushed to Amazon S3 for easy retrieval. Lifecycle policies enabled on the S3 bucket to archive and remove old backups.
  • Infrastructure: All components are provisioned and managed using Terraform, including the Kubernetes cluster, load balancers, rest of the architecture setup.
  • Additional Components: Integrated security using AWS Secrets Manager, hosted within an Amazon VPC with CI/CD pipelines deployed via GitHub Actions for continuous integration and delivery.
